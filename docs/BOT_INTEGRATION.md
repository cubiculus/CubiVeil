# Интеграция Telegram-бота с утилитами CubiVeil

## Обзор

CubiVeil Bot теперь интегрирован со всеми утилитами проекта, предоставляя полный контроль над сервером через Telegram.

## Установка

После запуска `install.sh` выполните установку бота:

```bash
bash setup-telegram.sh
```

Бот будет автоматически настроен с доступом ко всем утилитам.

## Команды бота

### 📊 Мониторинг

#### `/status`
Показать краткий статус сервера:
- Загрузка CPU
- Использование RAM
- Использование диска
- Uptime
- Количество активных пользователей

#### `/monitor`
Полный снимок состояния сервера через `utils/monitor.sh`:
- Системные ресурсы (CPU, RAM, Disk)
- Статус всех сервисов
- Сетевая информация (внешний и внутренний IP)
- Количество активных пользователей
- Последние события из логов

Результат отправляется как файл.

#### `/services`
Статус всех сервисов:
- 🟢 Sing-box Core
- 🟢 CubiVeil Bot
- 🟢 Firewall (UFW)
- 🟢 Fail2ban
- 🟢 Health Check

#### `/logs [service] [lines]`
Получить последние логи сервиса.

**Использование:**
- `/logs sing-box 50` — последние 50 строк логов Sing-box
- `/logs cubiveil-bot 10` — последние 10 строк логов бота

**Доступные сервисы:** `sing-box`, `cubiveil-bot`

#### `/alerts`
Статус алертов и текущие значения:
- CPU и порог превышения
- RAM и порог превышения
- Диск и порог превышения
- Активные алерты (если есть)

---

### 💾 Бэкапы

#### `/backup`
Создать полный бэкап системы через `utils/backup.sh`:
- Sing-box (конфигурация)
- SSL сертификаты
- Ключи шифрования
- Системная информация

Бэкап отправляется как файл.

**Время выполнения:** ~2-5 минут в зависимости от размера базы данных.

#### `/backups`
Список доступных бэкапов с информацией:
- Имя бэкапа
- Размер
- Дата создания

---

### 👥 Пользователи

#### `/users`
Список всех пользователей через `utils/manage-profiles.sh`:
- Имя пользователя
- Статус (active/disabled/expired)
- Использованный трафик
- Срок действия

#### `/adduser`
Инструкция по добавлению пользователя через SSH.

Добавление пользователя требует интерактивного ввода и выполняется через SSH:

```bash
bash /opt/cubiveil/utils/manage-profiles.sh add
```

#### `/qr <username>`
Сгенерировать QR-код для подключения пользователя.

**Использование:**
- `/qr user1` — QR-код для пользователя user1

QR-код отображается прямо в Telegram (Unicode версия).

---

### 🔧 Управление

#### `/restart <service>`
Перезапустить сервис.

**Использование:**

- `/restart sing-box` — перезапустить Sing-box
- `/restart cubiveil-bot` — перезапустить бота
- `/restart all` — перезапустить все сервисы

**Примечание:** Перезапуск `cubiveil-bot` приведёт к разрыву соединения с ботом.

#### `/update`
Проверить наличие обновлений CubiVeil.

Проверяет текущую версию и последнюю версию с GitHub. Если обновление доступно, выводит инструкцию для обновления через SSH.

Обновление выполняется вручную через SSH:
```bash
bash /opt/cubiveil/utils/update.sh
```

#### `/export`
Экспорт конфигурации CubiVeil через `utils/export-config.sh`:

Конфигурация включает:
- Конфигурацию Sing-box
- Конфигурацию Sing-box
- SSL сертификаты
- Настройки сети

Экспортированный файл отправляется в JSON формате.

#### `/diagnose`
Запустить полную диагностику системы через `utils/diagnose.sh`:

Диагностика включает:
1. Проверка окружения
2. Проверка DNS
3. Проверка SSL сертификата
4. Проверка соединений
5. Проверка сервисов
6. Проверка портов
7. Анализ логов
8. Проверка ресурсов
9. Генерация отчёта
10. Рекомендации

Отчёт отправляется как файл.

**Время выполнения:** ~1-2 минуты.

## Автоматизация через Cron

### Ежедневный отчёт

Отчёт отправляется автоматически по расписанию (по умолчанию в 09:00 UTC).

Содержит:
- Статус ресурсов (CPU, RAM, Disk)
- Uptime
- Количество активных пользователей
- Бэкап базы данных

### Алерты

Алерты проверяются каждые 15 минут и отправляются при превышении порогов.

Пороги настраиваются при установке бота:
- `ALERT_CPU` — по умолчанию 80%
- `ALERT_RAM` — по умолчанию 85%
- `ALERT_DISK` — по умолчанию 90%

Алерт отправляется только один раз при переходе из нормы в превышение, чтобы избежать спама.

## Безопасность

### Авторизация

Бот принимает команды только от одного `CHAT_ID`, указанного при установке.

### Ограничения доступа

Systemd сервис бота имеет следующие ограничения:
- `ProtectHome=true` — нет доступа к домашним директориям
- `ProtectSystem=strict` — только чтение системных файлов
- `NoNewPrivileges=true` — нет дополнительных привилегий
- `ReadWritePaths` — запись только в `/opt/cubiveil-bot/`
- `ReadOnlyPaths` — чтение базы данных Sing-box

### Шифрование

Утилиты `backup.sh` поддерживают шифрование бэкапов через `age`. Бэкапы отправляемые ботом не шифруются (ограничение Telegram).

Для зашифрованных бэкапов используйте SSH.

## Устранение неполадок

### Бот не отвечает

1. Проверьте статус сервиса:
   ```bash
   systemctl status cubiveil-bot
   ```

2. Перезапустите бота:
   ```bash
   systemctl restart cubiveil-bot
   ```

3. Проверьте логи:
   ```bash
   journalctl -u cubiveil-bot -n 50
   ```

### Утилиты не выполняются

1. Проверьте путь к утилитам:
   ```bash
   ls -la /opt/cubiveil/utils/
   ```

2. Проверьте переменную окружения в systemd:
   ```bash
   systemctl show cubiveil-bot -p Environment
   ```

3. Проверьте права доступа:
   ```bash
   chmod +x /opt/cubiveil/utils/*.sh
   ```

### Команда работает долго

Некоторые команды занимают много времени:
- `/backup` — 2-5 минут
- `/diagnose` — 1-2 минуты
- `/monitor` — 10-30 секунд

Бот отправляет уведомление о начале выполнения и результата.

## Разработка и расширение

### Добавление новой команды

1. Добавьте функцию в `assets/telegram-bot/commands.py`:
   ```python
   def _newcommand(self):
       """Описание команды"""
       self.telegram.send("Ответ команды")
   ```

2. Добавьте обработчик в метод `handle`:
   ```python
   elif cmd == "/newcommand":
       self._newcommand()
   ```

3. Перезапустите бота:
   ```bash
   systemctl restart cubiveil-bot
   ```

### Интеграция с новой утилитой

1. Убедитесь, что утилита находится в `utils/`
2. Используйте функцию `run_utility` в боте:
   ```python
   returncode, stdout, stderr = run_utility("newutil", args, timeout=120)
   ```

3. Обработайте результат:
   ```python
   if returncode == 0:
       self.telegram.send(f"✅ {stdout}")
   else:
       self.telegram.send(f"❌ {stderr}")
   ```
